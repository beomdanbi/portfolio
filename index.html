Stacks

프로젝트에 필요한 기술들을 빠르게 습득하여 적용해 나갑니다. 자신의 스타일만 고수하지 않고 타인과의 협업 역시 중요하게 생각합니다.
기술 구분 로고 활용 수준 평가
ASP.NET, .NET .NET으로 주로 프로젝트를 진행해 왔습니다. Webform 및 .Net(MVC)로 프로젝트 초기 구성부터 운영까지 모듈화된 프로그램 개발이 가능합니다.
심플한 개발을 선호하며, 자사 및 타사와의 연동은 SOAP WebService, Rest Api로 개발하였습니다.
더 나은 프로젝트 구조를 위해 항상 고민하고 있습니다.
JavaScript JavaScript, Jquery는 처음 접한 후 서버 컨트롤과는 다른 반응성에 반해 즐겨사용하고 있습니다.
업무 특성상 하위 운영체제와 브라우저간의 호환성이 늘 중요했기에, IE 구버전 부터 크롬 브라우저까지 모두 정상 작동하도록 개발하고 있습니다.
최근에는 TypeScript 의 유지보수성 증가과, 코드 신뢰도 증가에 매력을 느껴 업무에 적용 시키고 있습니다.
React 업무상 필요로 인하여 REACT를 접하였고, SPA 프로그램에 대해서 생각하는 계기가 되었습니다.
성능 이슈가 있던 프로그램을 개선하는 과정에서 많은 공부가 되었고, 앞으로 더 알아가고자 하는 분야입니다.
Spring 주로 .NET으로 개발을 진행했고 토이프로젝트로 안드로이드앱만 접하다가, 스프링에 흥미를 느껴 개인적으로 공부를 시작하게 되었습니다.
그러던 중 업무상 필요로 스프링 백엔드 개발을 하게되었습니다.
.NET으로 개발된 작은 프로젝트들을 스프링으로 전환하는 프로젝트를 진행중입니다. 흥미로운 작업이고 개인 역량 증진에 큰 도움이 될 것이라 생각합니다.
Node.js Node.js로 Express 서버 구성이 가능합니다. MSSQL 및 기타 웹서비스와 연동하여 데이터 제어가 가능하고 SOCKET.IO/ MQTT를 이용한 통신 개발을 진행했습니다.
소켓 서버 구성을 위해 처음 접하였으나 비동기 서버의 작동 방식에 새로움을 느꼈습니다.
MSSQL 관계형 데이터베이스를 설계하고 RDMBS를 사용하여 높은 생산성으로 프로시저 및 함수를 작성합니다. 정규화, 조인, 인덱스, 쿼리튜닝 등 올바른 DB튜닝을 하고자 합니다.
무엇보다 데이터의 정합성을 최우선으로 생각하고 있습니다. 올바른 데이터의 생성을 위한 로직 구성을 위해 최선을 다합니다.
ORACLE MSSQL 과 마찬가지로 관계형 데이터베이스를 설계하고 사용하여 높은 생산성으로 프로시저 및 함수를 작성할 수 있습니다. RDBMS의 종류에 따라, SQL 문법이 조금씩 상이하나 개발 투입시 단기간에 퍼포먼스를 끌어올릴 수 있습니다.
다만, 주로 DBA 영역인 DB장애 관련한 부분들이나 모니터링 부분들은 부족한 부분이 많다고 느끼고 있습니다. 앞으로 꾸준히 노력하여 기술 능력을 업그레이드 하고자 합니다.
HTML5&CSS3 웹 개발의 기본이 되는 HTML,CSS는 즉시 사용이 가능힙니다. 기본적인 퍼블리싱이 가능하며, 웹과 모바일 모두 작업 가능합니다.
bootstarap을 주로 이용하며, fontawesome과 googleapis의 심플한 아이콘을 사용하여 디자인하고 있습니다.
제가 생각하고 이해하는 주관적인 평가입니다.
